# Mordomo — MEMORY

Este arquivo aponta onde Mordomo deve buscar memória/contexto. Ele não armazena secrets nem dados vivos.

## Boot mínimo

1. `memories/current.md` — camada quente/current.
2. `memories/pending.md` — pendências.
3. `memories/decisions.md` — decisões permanentes.
4. `memories/lessons.md` — lições.
5. `areas/operacoes/rotinas/runtime-profile-channel-inventory-2026-05-19.md` — perfis/canais/status documental.

## Por domínio

- LK: `memories/lk.md`, `areas/lk/MAPA.md`, `agentes/lk/`.
- Zipper: `memories/zipper.md`, `areas/zipper/MAPA.md`, `agentes/zipper/`.
- SPITI: `memories/spiti.md`, `areas/spiti/MAPA.md`, `agentes/spiti/`.
- Operações: `areas/operacoes/MAPA.md`, `empresa/rotinas/_index.md`, `empresa/skills/_index.md`.

## Regra de escrita

Registrar no Brain apenas o que for durável: decisão, rotina, skill, pendência, lição, contexto estável ou receipt. Não transformar cada mensagem transitória em memória permanente.
