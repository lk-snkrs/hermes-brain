# Reports de Governança

Pasta para relatórios curados que servem como evidência de governança do Hermes Brain.

## Critério de entrada

Entram aqui apenas relatórios `.md` que documentam decisões, auditorias, controles ou verificações estruturais que precisam continuar auditáveis após compactações de contexto.

Exemplos permitidos:

- auditorias de memória/decisões;
- evidências de health check e guardrails;
- relatórios de governança Brain/Bruno/OpenClaw;
- sínteses executivas curadas, sem payload bruto.

## O que não entra

- logs brutos;
- HTML/CSV/exports;
- dumps de APIs;
- receipts operacionais com PII;
- credenciais, secrets, payloads sensíveis;
- relatórios gerados automaticamente sem curadoria.

## Regra operacional

Se um relatório bruto for importante, criar uma síntese `.md` curada nesta pasta ou na área correspondente. O bruto continua local/privado.
