# Hermes Learning Ledger — 2026-06-20

## Padrão aprendido

O watchdog de Memory OS não deve registrar auto-heal quando o arquivo final já está idêntico ao contrato desejado. Auto-heal sem mudança real cria ruído recorrente, backup desnecessário e reduz a qualidade do sinal do 02h.

## Autonomia

Classificação: **A1 local/script-safe**.

Permitido executar: patch idempotente em script local, `py_compile`, execução JSON do checker, Brain Health, relatório local, receipt via writer.

Não permitido neste cron: alterar `/opt/data/config.yaml`, mudar provider de memória, reiniciar gateway, alterar Docker/VPS/Traefik, criar/alterar cron, escrever em sistemas externos ou tocar secrets.

## Resultado

- `generator_auto_heal` passou a retornar `status=ok`, `changed=false`, `reason=contract_current` quando o contrato já está atual.
- O alerta global de Memory OS permanece `action_required` por provider guard, que é decisão/config separada.

## Próxima melhoria segura

Separar explicitamente, em pacote posterior, a semântica do guard de provider de memória: Mem0/provider externo proibido vs Honcho configurado no runtime. Qualquer alteração de runtime/config precisa de aprovação escopada e rollback.
