# Hermes Learning Ledger — 2026-05-30

## Aprendizados do Dia

### 1. EXPECTED_VERSION_FRAGMENT deve ser atualizado após mudança de runtime
- **Origem:** Watchdog `edd06fe19397` alertando `versão inesperada: esperado v0.14.0; obtido v0.15.1` todos os dias
- **Causa raiz:** Script não foi atualizado quando launcher `/opt/hermes/hermes` foi modificado para usar `hermes-15-install`
- **Destino:** `/opt/data/scripts/hermes_runtime_cron_watchdog.py` linha 26
- **Ação aplicada:** `EXPECTED_VERSION_FRAGMENT = "v0.14.0"` → `"v0.15.1"` (A1, local, read-only behavior)
- **Verificação:** watchdog rodou silencioso após fix
- **Regra futura:** Sempre que o runtime mudar de versão (via launcher ou imagem), atualizar `EXPECTED_VERSION_FRAGMENT` no mesmo ciclo. Adicionar ao checklist de upgrade.

### 2. EXPECTED_GATEWAY_HOMES deve distinguir REQUIRED vs OPTIONAL_DORMANT
- **Origem:** Alertas diários para `/opt/data/profiles/lk-ops`, `lk-shopify`, `lk-trends` que são perfis preparados mas intencionalmente não iniciados
- **Causa raiz:** Watchdog tratava todos os perfis da allowlist como required-running
- **Destino:** Script watchdog — nova distinção `REQUIRED_GATEWAY_HOMES` vs `OPTIONAL_DORMANT_GATEWAY_HOMES`
- **Ação aplicada:** Split implementado; missing check usa apenas `REQUIRED_GATEWAY_HOMES`
- **Verificação:** watchdog silencioso (rc=0) após fix
- **Regra futura:** Quando um novo perfil é ativado (aprovação scoped + launcher), mover de `OPTIONAL_DORMANT` para `REQUIRED`. Quando desativado intencionalmente, mover para `OPTIONAL_DORMANT`.

### 3. SystemExit(75) do gateway = /restart command, não crash anômalo
- **Origem:** gateway-exit-diag.log mostrando múltiplos `SystemExit code=75` às 01:10-01:27 UTC
- **Diagnóstico:** `.restart_last_processed.json` confirma `/restart` via Telegram às 01:27:31 UTC; exit(75) é `GATEWAY_SERVICE_RESTART_EXIT_CODE` intencional
- **Container:** recuperou em 01:31 UTC, estável por 3+ horas
- **Regra futura:** Ao triar restart loops no gateway-exit-diag, verificar primeiro `.restart_last_processed.json` antes de escalar. Se recovery ocorreu em <10min e não há crashes após recovery, classificar como "incidente resolvido" não como loop ativo.

### 4. Anthropic OAuth tokens expiram; pool rotaciona para fallback automaticamente
- **Origem:** 401 em `anthropic-oauth-2` e `ANTHROPIC_TOKEN` às 05:00 UTC; pool rotacionou para OpenRouter
- **Impacto:** Serviço funcionando via fallback; sem interrupção de Telegram/cron
- **Risco:** Se OpenRouter também falhar, haverá degradação. Renovação preventiva recomendada.
- **Regra futura:** Anthropic OAuth tokens devem ser renovados periodicamente (especialmente após longas sessões sem uso ou após mudanças de runtime). Adicionar ao checklist mensal.

### 5. Observability helper captura snapshots pontuais, não estado contínuo
- **Origem:** Helper às 04:30 capturou container em `Restarting`; às 05:00 API /health = ok
- **Regra futura:** Ao receber alert de container `Restarting` do helper, verificar API /health e agent.log antes de escalar. Se /health responde e logs mostram atividade recente, classificar como transiente/recuperado.
