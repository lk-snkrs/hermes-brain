# Hermes Learning Ledger — 2026-06-05

## 1. wacli JSON em cron não é necessariamente stdout puro

- **Origem:** cron vivo `LK Pulso Comercial 16h read-only delivery` falhou em 2026-06-04 com `JSONDecodeError` ao executar envio WhatsApp aprovado via `wacli --json`.
- **Padrão aprendido:** scripts de cron não devem fazer `json.loads(proc.stdout)` diretamente para `wacli --json`; stdout pode trazer ruído/banners/control text antes do objeto JSON.
- **Ação aplicada:** `/opt/data/scripts/lk_report_external_delivery.py` recebeu parser resiliente que lê a última linha/objeto com aparência JSON e scrubba stdout inválido.
- **Destino durável:** skill `wacli-whatsapp-cli` atualizada na seção de segurança/regras de script.
- **Verificação:** `py_compile` OK; teste local com stdout ruidoso OK; dry-run `pulse-finance-16h` sem `--send` retornou `dry_run_ok`.
- **Autonomia:** A1 local/script; não executou novo envio externo.
- **Próxima regra:** testar entrega aprovada só por dry-run quando o bug for parser; confirmação de envio fica para o próximo run real ou aprovação explícita.
