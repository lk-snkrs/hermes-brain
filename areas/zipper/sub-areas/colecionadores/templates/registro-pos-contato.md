# Template — Registro Pós-contato com Colecionador

## Objetivo

Padronizar o registro interno depois de um contato aprovado com colecionador, preservando histórico real, tom Zipper e próximos passos sem inventar preferência ou intenção de compra.

## Pré-condição obrigatória

O contato externo precisa ter sido previamente aprovado por Lucas/Osmar. Se não houve aprovação, este template não deve ser usado para justificar envio retroativo.

## Entrada mínima

```text
Colecionador:
Responsável pelo contato:
Data/hora:
Canal usado:
Obra/artista mencionado:
Mensagem aprovada por:
Link/registro do rascunho aprovado:
```

## Resultado do contato

```text
Status:
- [ ] Sem resposta ainda
- [ ] Respondeu com interesse
- [ ] Pediu mais informações
- [ ] Pediu reserva/proposta
- [ ] Recusou
- [ ] Encaminhar para Lucas/Osmar
- [ ] Outro: [descrever]

Resumo factual da resposta:
Próximo passo combinado:
Prazo:
Responsável:
```

## Leitura comercial segura

```text
O que sabemos:
- 

O que parece provável, mas precisa confirmar:
- 

O que não podemos afirmar:
- preferência não declarada
- orçamento/poder de compra não informado
- intenção de compra não confirmada
```

## Próximo rascunho, se necessário

```text
Objetivo do follow-up:
Tom:
Dados/obras que precisam ser confirmados:
Aprovação necessária:
```

## Regras

- Não transformar silêncio em desinteresse definitivo sem contexto.
- Não registrar inferência psicológica ou financeira como fato.
- Não usar tom de pressão, urgência artificial ou liquidação.
- Sempre manter a narrativa como curadoria/pertencimento/identidade cultural.

## Onde registrar

- CRM/fonte viva definida por Lucas/Osmar quando disponível.
- `areas/zipper/contexto/lessons.md` apenas para aprendizados recorrentes e não sensíveis.
- Nunca registrar dados pessoais sensíveis em docs versionados sem necessidade e aprovação.
