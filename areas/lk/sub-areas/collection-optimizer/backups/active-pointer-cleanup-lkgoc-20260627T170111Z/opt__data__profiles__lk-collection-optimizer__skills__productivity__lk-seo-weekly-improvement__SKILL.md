## Common Pitfalls

1. **Relatório solto.** SEO só vale se virar fila de melhoria de página/PDP.
2. **Confundir auditoria com write.** Audit e preview são livres; writes precisam aprovação.
3. **Auditar tudo e não priorizar nada.** Weekly loop deve escolher poucas ações de alto impacto.

## Shopify source-page rendering pitfalls

- For LK source pages/guides, never assume a patched snippet is the active storefront renderer. Check the page `template_suffix`, then `templates/page.<suffix>.json`, then the `sections.main.type` section asset before concluding a CSS/layout fix worked.
- If Lucas reports visual misalignment after a successful Shopify PUT/readback, prioritize DOM measurement of hero/section widths and active template/section readback over adding more generic CSS.
- For hero/body width parity, the section container should match the hero rail (commonly 1120px) and direct editorial children should be widened deliberately (`max-width:100%; width:100%`) if older 860px readability rules are making the block look smaller than the hero.
- See `references/shopify-source-page-template-section-width-parity-20260526.md` for the Moon Shoe cache/template/section debugging pattern.
- See `references/shopify-source-page-hero-only-and-collection-top-area-20260526.md` for the follow-up lesson: keep dark styling scoped to the hero only, return all guide/body sections to light, verify alignment visually/computed in browser, and QA collection top description/photos as well as the lower guide/FAQ.
