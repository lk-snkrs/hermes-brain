# LKGOC — PRD operacional

LKGOC = LK Growth Optimized Collection: sistema para transformar uma intenção/modelo prioritário em experiência comercial + editorial + SEO/GEO auditável.

## Objetivo

Criar coleções LK que vendem e são citáveis: produto-first, visual premium, guia dedicado, pesquisa factual, dados SEO, preview Shopify DEV e aprendizado pós-lançamento.

## Quando usar LKGOC Full

- modelo/collab com demanda ou potencial comercial relevante;
- coleção que Lucas quer elevar ao padrão 204L;
- lançamento estratégico, tendência ou oportunidade GSC/DataForSEO;
- correção de coleção que saiu fora do padrão.

## Quando NÃO usar Full

- coleção operacional pequena sem guia dedicado;
- página sem produtos suficientes e sem prioridade comercial;
- ajuste pontual de title/meta sem mudança visual/editorial.

Nesses casos usar LKGOC Lite ou Correção LKGOC, declarando o nível.

## Níveis

- **Full:** coleção 204L + guia Moon Shoe + evidence packet completo + DEV preview + QA + score + impact review.
- **Lite:** mantém produto-first, pesquisa web e Claude SEO; pode dispensar Ahrefs/guia longo se baixa prioridade, mas precisa justificar.
- **Correção:** alinhar coleção/guia existente ao canônico, com antes/depois e QA visual.
- **Não-LKGOC:** coleção simples; não chamar de LKGOC.

## Sucesso

- aprovação visual de Lucas sem drift;
- coleção com score LKGOC >= 85 antes de produção;
- nenhuma produção sem approval atual;
- impacto D+7/D+14/D+30 medido quando publicado.
