# REGRA BLOQUEANTE — LKGOC Visual Spec: lifestyle, FAQ 2 colunas e referências de moda

Registrado em: 2026-06-06T14:35:53.273747Z

Motivo: falha operacional em 2026-06-06. Guias foram gerados com imagem de produto isolado, FAQ em 1 coluna e sem referências editoriais/de veículos de moda, apesar de o padrão aprovado exigir experiência editorial premium.

## Bloqueio
Nenhuma página/guia/coleção pode ser chamada de LKGOC se faltar qualquer item abaixo:

1. **Hero lifestyle/on-foot/editorial**
   - Proibido usar foto packshot/produto isolado como imagem hero principal.
   - Hero deve usar pessoa usando o produto, imagem editorial, campanha, street style ou referência visual equivalente.
   - Se não houver imagem editorial segura/licenciada/aprovada, parar e pedir asset ao Lucas; não improvisar com packshot.

2. **FAQ visual em 2 colunas no desktop**
   - FAQ deve bater o padrão visual aprovado: grid/duas colunas em desktop, responsivo no mobile.
   - FAQ em coluna única só é aceitável em mobile.

3. **Referências editoriais e veículos de moda**
   - A página deve conter bloco de contexto com referências/menções editoriais relevantes quando aplicável: Vogue, GQ, Highsnobiety, Hypebeast, Who What Wear, Elle, Harper's Bazaar, Sneaker Freaker, Complex, etc.
   - Referências precisam ser verificadas ou marcadas como “contexto editorial”, sem inventar citação factual específica.

4. **Gold source visual obrigatório**
   - Antes de qualquer write, comparar contra screenshot da página aprovada pelo Lucas.
   - Checklist visual deve ser preenchido antes do write: hero, FAQ, media references, produto grid, CTA, typography, spacing.

5. **Não publicar Page global sem aprovação**
   - Guias Shopify Page são customer-facing se publicados; preview por tema não isola Page global.
   - Para QA, usar dev theme/asset/template ou page despublicada quando possível. Publicar Page exige aprovação explícita se ficar acessível em production.

## Definition of Done adicional
- Screenshot mobile e desktop lado a lado com gold source aprovado.
- Verificação visual humana antes de replicar para lote.
- Se Lucas apontar divergência estrutural, executar rollback antes de nova tentativa.
