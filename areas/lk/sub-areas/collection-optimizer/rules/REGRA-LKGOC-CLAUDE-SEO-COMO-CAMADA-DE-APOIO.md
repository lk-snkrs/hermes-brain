# REGRA — LKGOC usa Claude SEO como camada de apoio

Registrado em: 20260609T121055Z
Status: **CANÔNICO / OPERACIONAL**

## Decisão Lucas

Para coleções LKGOC, usar Claude SEO como camada especialista de apoio para conteúdo, intenção de busca, FAQ, GEO/AI Search e validação semântica.

Claude SEO **não é dono do LKGOC** e não pode substituir o padrão visual 204L, o template Shopify padrão ou a aprovação de Lucas.

## Papel do Claude SEO no LKGOC

Usar para:

- interpretar intenção de busca;
- analisar SERP, concorrentes e lacunas;
- sugerir estrutura de title/meta;
- sugerir perguntas FAQ baseadas em dúvidas reais;
- validar se o guia está claro, específico e citável;
- melhorar legibilidade GEO/AI Search;
- apontar genericidade, claims fracos e oportunidades semânticas;
- apoiar blocos citáveis para ChatGPT/Perplexity/Gemini/AI Overviews.

Não usar para:

- criar layout novo;
- mudar shell visual 204L;
- escolher imagem hero sem media manifest;
- priorizar coleção sem dados comerciais/GSC/Shopify quando a decisão exigir decision-grade;
- inventar fatos, tendências, materiais, datas ou contexto sem fonte;
- publicar, promover ou fazer write em Production;
- contrariar guardrails LK.

## Ordem operacional

1. LKGOC Intake / PRD / perguntas.
2. Evidence packet com fontes reais.
3. Claude SEO como camada de análise e validação.
4. Text packet no tom LK.
5. Media manifest.
6. Build no template Shopify LKGOC em DEV/unpublished.
7. QA visual 204L + readback.
8. Approval Lucas antes de Production.

## Regra de decisão

Claude SEO é worker especialista. O aceite final continua dependente de:

- Gold Source 204L;
- template `collection.lkgoc`;
- QA visual desktop/mobile;
- evidence/text/media packets;
- aprovação Lucas para Production.
