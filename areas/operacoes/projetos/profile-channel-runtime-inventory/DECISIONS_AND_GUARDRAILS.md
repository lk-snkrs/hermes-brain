# Hermes Profile / Channel Runtime Inventory — Decisions and Guardrails

## Decisão Brain OS

Criar um hub canônico para reduzir ambiguidade entre documentação histórica, configuração, runtime vivo e ação externa.

## Guardrails

1. Documento de profile/canal não prova ativação runtime; verificar Hermes/gateway/plataforma antes de afirmar que está ativo.
2. Não criar, reiniciar, pausar ou reconfigurar profile, bot, gateway, toolset ou canal como parte deste hub.
3. Qualquer mudança real em runtime/profile/gateway/canal exige aprovação escopada, backup/rollback e verificação pós-ação.
4. Inventário deve separar documented, configured, active e verified.
