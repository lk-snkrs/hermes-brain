# Nomenclatura — Cérebro Cimino vs Hermes Brain

Status: **canônico**
Atualizado: 2026-05-22

## Decisão curta

**Cérebro Cimino** e **Hermes Brain** são o mesmo sistema lógico: a grande mente operacional de Lucas Cimino.

A diferença é apenas de nome/contexto:

- **Hermes Brain**: nome técnico/documental do repositório e da camada de memória/rotinas/skills.
- **Cérebro Cimino**: nome humano/empresarial para o mesmo cérebro quando falamos da operação do Lucas e das empresas.
- **Hermes Agent**: o agente/runtime que conversa no Telegram e usa ferramentas. Ele **não é** o cérebro; ele lê, escreve e executa usando o Brain.
- **Perfis/agentes especialistas**: Mordomo, LK Growth, SPITI e outros são braços/áreas do sistema, não cérebros separados.

```text
Lucas / Telegram
  ↓
Hermes Agent (runtime e ferramentas)
  ↓
Cérebro Cimino = Hermes Brain = Grande Mente / Hermes COO
  ├── Lucas pessoal / Mordomo

  └── Governança / Segurança / Aprovações
  ↓
Dados vivos: APIs, bancos, Shopify, GMC, email, crons, logs e documentos
```

## Regra operacional

Quando houver dúvida entre “Cérebro Cimino” e “Hermes Brain”, tratar como sinônimos e usar este padrão:

1. Em arquivos técnicos, relatórios e skills: usar **Hermes Brain**.
2. Em explicações executivas para Lucas: pode usar **Cérebro Cimino / Hermes Brain** na primeira menção.
3. Não criar repositório, profile, memória ou rotina duplicada só por causa da diferença de nome.
4. Não copiar estruturas OpenClaw literalmente; adaptar para Hermes-native.

## Anti-confusão

- Não existe um “Cérebro Cimino” separado concorrendo com o Hermes Brain.
- Não existe um “Hermes Brain” separado do COO/Grande Mente do Lucas.
- Se um documento antigo sugerir separação, ele deve ser corrigido para apontar para esta nomenclatura.
- Dados vivos continuam sendo fonte de verdade para fatos operacionais; o Brain guarda contexto, decisões, rotinas e evidências curadas.
